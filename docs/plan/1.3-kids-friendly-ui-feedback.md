# 1.3 — Kids-Friendly UI Feedback

> This document follows `docs/WRITING_GUIDELINES.md`.

## Status

Complete

## Phase Type

UI / UX Polish / Child-Friendly Feedback

---

## Objective / Tujuan

### Indonesia

Meningkatkan tampilan dan pengalaman bermain **Animal Food** agar lebih ramah untuk anak usia 3 tahun.

Phase ini berfokus pada:

* UI yang lebih lucu, lembut, dan mudah disentuh
* feedback benar/salah yang lebih jelas dan menyenangkan
* layout mobile portrait yang lebih rapi
* visual reward sederhana setelah jawaban benar
* pengalaman bermain yang lebih nyaman tanpa mengubah core gameplay

### English

Improve the look and play experience of **Animal Food** so it feels more friendly for 3-year-old children.

This phase focuses on:

* cuter, softer, and easier-to-touch UI
* clearer and more delightful correct/wrong feedback
* cleaner mobile portrait layout
* simple visual rewards after correct answers
* a smoother play experience without changing the core gameplay

---

## Background / Latar Belakang

### Indonesia

Phase 1.2 sudah membuat **Animal Food** playable.

Player sudah bisa:

* membuka Animal Food
* melihat animal dan food options
* drag food ke animal
* menerima feedback benar/salah
* lanjut level
* masuk ke ResultScene setelah semua level selesai

Namun, pengalaman visual dan UX masih perlu dipoles agar lebih cocok untuk anak kecil.

Phase 1.3 tidak membuat game baru. Phase ini hanya meningkatkan kualitas experience dari gameplay yang sudah ada.

### English

Phase 1.2 made **Animal Food** playable.

The player can already:

* open Animal Food
* see animal and food options
* drag food to the animal
* receive correct/wrong feedback
* progress through levels
* reach ResultScene after completing all levels

However, the visual experience and UX still need polish so the game feels more suitable for young children.

Phase 1.3 does not create a new game. This phase only improves the experience quality of the existing gameplay.

## Important Rules / Aturan Penting

### Indonesia

Jangan mengubah core gameplay yang sudah selesai di Phase 1.2.

Pertahankan aturan berikut:

* tidak ada timer
* tidak ada lives
* tidak ada punishment
* tidak ada game over
* tidak ada kata negatif
* tidak ada layout yang terlalu padat
* tidak ada tombol kecil
* tidak ada feedback kasar

Correct answer harus terasa menyenangkan, tetapi tidak berlebihan.

### English

Do not change the core gameplay completed in Phase 1.2.

Keep these rules:

* no timer
* no lives
* no punishment
* no game over
* no negative wording
* no crowded layout
* no small buttons
* no harsh feedback

Wrong answers must still feel safe and gentle.

Correct answers should feel delightful, but not excessive.

## Visual Direction / Arah Visual

### Indonesia

Gunakan visual yang terasa:

* cute
* soft
* playful
* warm
* friendly
* clean
* tidak terlalu ramai

Gunakan pendekatan visual:

```txt
Background: warm cream / soft pastel
Cards: rounded, large, soft shadow
Buttons: large, rounded, clear
Animal area: large and centered
Food options: big and easy to drag
Feedback: friendly bubble
Reward: small stars/confetti/bounce
```

Hindari:

* warna terlalu gelap
* warna terlalu neon
* terlalu banyak elemen dalam satu layar
* animasi terlalu cepat
* teks terlalu panjang
* ukuran touch target terlalu kecil

### English

Use visuals that feel:

* cute
* soft
* playful
* warm
* friendly
* clean
* not too crowded

Use this visual direction:

```txt
Background: warm cream / soft pastel
Cards: rounded, large, soft shadow
Buttons: large, rounded, clear
Animal area: large and centered
Food options: big and easy to drag
Feedback: friendly bubble
Reward: small stars/confetti/bounce
```

Avoid:

* overly dark colors
* overly neon colors
* too many elements on one screen
* overly fast animation
* long text
* touch targets that are too small
