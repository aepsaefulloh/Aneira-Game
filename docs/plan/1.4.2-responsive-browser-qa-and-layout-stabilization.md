## Canvas and Safe Area

The game should handle different viewport sizes without clipping important UI.

Rules:

```txt
- Keep important UI inside safe margins.
- Use scene width and height dynamically.
- Avoid hardcoded center positions that only work on one screen size.
- Avoid placing controls too close to screen edges.
- Keep header height proportional.
- Keep bottom interaction area comfortable.
```

Recommended safe margin:

```txt
Mobile: 16–24px
Tablet: 24–32px
Desktop: 32–48px
```

In Phaser, prefer deriving positions from:

```ts
const { width, height } = this.scale
```

or existing project helpers if available.

# Part 5 — UI Control Proportion Rules

## Language Selector

The language selector should be easy to tap but not visually dominant.

Recommended pattern:

```txt
[ IDN | EN ]
```

Rules:

```txt
- Use pill style.
- Keep it in the settings/header area.
- Do not place it near the drag-and-drop food area.
- Active language should be clear.
- Inactive language should remain readable.
- On small mobile screens, allow it to stack with sound toggle if needed.
```

Recommended minimum touch area:

```txt
44px height
```

---

## Sound Toggle

The sound toggle should feel like a secondary setting, not a main CTA.

Recommended pattern:

```txt
Sound On
Sound Off
```

or:

```txt
Audio On
Audio Off
```

Rules:

```txt
- Keep it visually smaller than primary buttons.
- Keep it large enough for touch.
- Place it in header/settings area.
- Do not place it too close to food cards.
- Do not let it dominate HomeScene or AnimalFoodScene.
```

Recommended minimum touch area:

```txt
44px height
```

# Part 6 — Scene-Specific Layout Rules

## HomeScene

HomeScene should prioritize:

```txt
1. app title
2. friendly subtitle
3. start button
4. language selector
5. sound toggle
```

Responsive behavior:

```txt
Mobile:
- settings group can stack vertically or wrap
- title remains centered
- start button is large and clear

Tablet:
- settings group can stay horizontal
- content uses centered max-width feeling

Desktop:
- avoid stretching content too wide
- keep menu content centered
```

Suggested layout:

```txt
┌─────────────────────────────┐
│        [IDN | EN] [Sound]    │
│                             │
│        Aneira Game           │
│   Mini games for little kids │
│                             │
│          [ Start ]           │
└─────────────────────────────┘
```

---

## AgeSelectScene

Age cards should be readable and touch-friendly.

Responsive behavior:

```txt
Mobile:
- cards stack vertically
- each card has enough height
- Age 3 card is clearly active
- Age 4 and Age 5 are disabled but visible

Tablet:
- cards can use 2-column or centered stack

Desktop:
- cards can be horizontal, but not too stretched
```

Suggested labels:

```txt
Age 3 — Animal Friends
Age 4 — Creative Discovery
Age 5 — Learning Adventure
```

---

## GameListScene

Game cards should follow the selected age theme.

Responsive behavior:

```txt
Mobile:
- game cards stack vertically
- available game is visually clear
- coming soon items are softer

Tablet:
- 2-column grid is acceptable

Desktop:
- centered grid with max content width
```

For Age 3:

```txt
Animal Food       → Available
Animal Sound      → Coming Soon
Color Match       → Coming Soon
Shape Match       → Coming Soon
```

---

## AnimalFoodScene

This is the most important responsive scene.

Priority order:

```txt
1. header must not dominate screen
2. instruction must be readable
3. animal target must be large enough
4. feedback bubble must not block dragging
5. food cards must be easy to drag
6. bottom area must not feel cramped
```

Responsive behavior:

```txt
Mobile portrait:
- keep 3 food cards visible if possible
- if width is too tight, reduce card spacing first, not card size too much
- maintain comfortable drag area
- avoid placing language selector in gameplay area
- sound toggle can stay in header

Tablet:
- animal target can be larger
- food cards can be wider
- spacing can increase

Desktop:
- center the game content
- avoid spreading food cards too far apart
```

Suggested layout:

```txt
┌─────────────────────────────┐
│ Back   Animal Food   Sound   │
│          Level 1 / 8         │
├─────────────────────────────┤
│ What does the rabbit eat?    │
│                             │
│        [Animal Target]       │
│                             │
│      [Feedback Bubble]       │
│                             │
│ [Food 1] [Food 2] [Food 3]   │
└─────────────────────────────┘
```

Recommended mobile constraints:

```txt
Header height: not more than 16–20% of screen height
Food card touch size: at least 88 x 88px if possible
Animal target: visually dominant, but not blocking food area
Bottom food area: always visible without scroll
```

---

## ResultScene

ResultScene should feel calm and celebratory.

Responsive behavior:

```txt
Mobile:
- success message centered
- replay and back buttons stacked
- celebration animation must not cover buttons

Tablet/Desktop:
- buttons can be side by side
- content remains centered
```
